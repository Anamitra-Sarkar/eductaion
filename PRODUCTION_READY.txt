DATABASE SCHEMA
================================================================================

Core Tables:
  1. users (email, hashed_password, role: admin/faculty/student)
  2. colleges (institution details)
  3. departments (CSE, ECE, MECH, CIVIL)
  4. students (30 seed records)
  5. subjects (code, name, faculty_id, credits)
  6. timetable_slots (day, time, room, conflicts detection)
  7. attendance_sessions (faculty-led sessions)
  8. attendance_records (student presence/absence/late)
  9. activities (workshops, seminars, hackathons, etc.)
  10. activity_enrollments (student participation)
  11. alumni (10 seed records with company tracking)

Relationships:
  - Users → Colleges (many-to-one)
  - Students → Users (one-to-one)
  - Students → Departments (many-to-one)
  - Subjects → Departments, Faculty (many-to-one)
  - Attendance → Students, Sessions (many-to-one)
  - Activities → Departments, Coordinator (many-to-one)
  - ActivityEnrollment → Students, Activities (many-to-one)
  - Alumni → Departments (many-to-one)

All relationships properly configured with cascade deletes.

================================================================================
API ENDPOINTS (40+)
================================================================================

AUTH:
  POST   /auth/register           Create user account
  POST   /auth/login              Get JWT token (returns access_token)
  GET    /auth/me                 Get current user

STUDENTS:
  GET    /students                List students (filters: dept, semester, status, search)
  GET    /students/{id}           Get student details
  POST   /students                Create student (admin only)
  PUT    /students/{id}           Update student (admin only)
  DELETE /students/{id}           Delete student (admin only)
  GET    /students/{id}/attendance-summary
  GET    /students/{id}/activity-summary

ATTENDANCE:
  GET    /attendance/sessions     List sessions (filters: date, subject, dept)
  POST   /attendance/sessions     Create session (faculty only)
  POST   /attendance/sessions/{id}/records  Mark attendance (bulk)
  PUT    /attendance/records/{id} Update single record
  GET    /attendance/report       Full attendance report (CSV-ready)
  GET    /attendance/defaulters   Students below attendance threshold
  GET    /attendance/heatmap      Attendance heatmap data

TIMETABLE:
  GET    /timetable               List slots (filters: dept, semester, day)
  POST   /timetable               Create slot (admin only)
  PUT    /timetable/{id}          Update slot
  DELETE /timetable/{id}          Delete slot
  GET    /timetable/conflicts     Check scheduling conflicts

ACTIVITIES:
  GET    /activities              List activities (filters: dept, type, status)
  POST   /activities              Create activity (faculty/admin)
  PUT    /activities/{id}         Update activity
  DELETE /activities/{id}         Delete activity
  POST   /activities/{id}/enroll  Enroll students (bulk)
  PUT    /activities/{id}/mark-attendance  Mark attendance

ALUMNI:
  GET    /alumni                  List alumni (filters: dept, batch, search)
  POST   /alumni                  Create alumni record
  PUT    /alumni/{id}             Update alumni
  DELETE /alumni/{id}             Delete alumni

ANALYTICS:
  GET    /analytics/dashboard-kpis       Today's KPIs
  GET    /analytics/attendance-trend     Historical trend
  GET    /analytics/department-comparison  Dept performance
  GET    /analytics/activity-completion   Activity stats

All endpoints return JSON, require JWT token, support filtering & pagination.
